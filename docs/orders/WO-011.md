# WO-011: 어항 뷰어 v1

## 배경

어항의 심장(러너)이 가동 중. 이제 창문이다. 뷰어는 "방금 일어난 일의 재생" 모델 — 이벤트 생성에 ~4분이 걸리므로, 발생 즉시가 아니라 log.jsonl에 새 항목이 커밋되는 순간 재생한다. v1의 캐릭터는 도형+이름표 (스프라이트는 WO-012). QA 목업의 화면 구성을 따른다: 상단 바 / 지도(주 영역) / 마을 로그 / 인물 카드.

## 작업 범위

### 1. 뷰어 서버 — `engine/viewer-server.mjs`

- 경량 로컬 서버 (외부 의존 최소): 정적 뷰어 페이지 서빙 + world/·log.jsonl을 JSON으로 제공 + 5초 폴링 또는 SSE로 갱신 통지. 포트 기본 4400, localhost 바인딩
- 격리 불필요 — 이 창은 신의 창이다. truth·텐션·비밀 전부 서빙한다. (공개 사이트 파이프라인과는 완전 별개 — deploy 쪽 격리는 무변경)

### 2. 뷰어 화면 — 목업 구성 그대로

**상단 바**: 미르한 · 기록력 날짜(clock.json 기준) · 현실 시각 · 날씨(최근 이벤트의 날씨 언급 또는 "맑음" 기본) · 다음 사건 예고(러너의 다음 예정 시각 — 러너가 `engine/runner-state.json`에 기록하게 하고 뷰어가 읽음)

**지도 (주 영역)**: places.json 기반 탑다운 배치 — 언덕(신전)·항구(빵집·대장간·시장·부두)· 갯벌. v1 비주얼: 단색 면 + 건물 rect + 라벨. 밤낮 톤 변화(현실 시각 기준 — 밤엔 어둡게, 창에 불). 캐릭터는 색 원 + 이름표:

- 앰비언트 루틴 (AI 없음): 인물별 간단한 일과표(시간대 → 장소: 집/일터/시장 등)를 `viewer/routines.json`으로 정의하고 그 사이를 걸어서 이동. 일과표 초안은 인물의 직업·quirks에서 Claude Code가 작성 (완료 보고에 첨부 — QA가 인물 정합 검수). 약간의 랜덤(배회, 멈춤)으로 반복감 완화
- 무명 배우: 회색 소형 원 + 역할 라벨("인부", "야경꾼") — staging에 등장할 때만 출현

**이벤트 재생**: log.jsonl 새 항목 감지 → staging 순서대로 배우 이동 → dialogue를 말풍선으로 순차 표시(발화당 4~6초) → 종료 후 루틴 복귀. 재생 중 다음 이벤트 도착 시 큐잉

**마을 로그 패널**: log_line 역순 피드. 마지막 열람 시점 이후 항목에 "새 소식" 구분선 — 못 본 시간 따라잡기가 이 패널의 존재 이유다. 항목 클릭 시 해당 이벤트 상세 (truth·public 대비 + dialogue 전문)

**인물 카드**: 지도나 로그에서 인물 클릭 → 카드: 이름·직업·구역·텐션, 비밀(truth 배지), 욕망(표면/숨김), wish(status), scars 목록, 관계(방향·type·tension), 등장 이벤트 타임라인. 사망·이주 인물은 도감에 남되 지도에서 제거

**신탁 패널 (미니)**: 접힌 패널 — 펼치면 텍스트 입력 + "공표 여부" 체크 + 내리기 버튼. oracles.json에 append까지만 (다음 이벤트에서 러너가 집어가는 기존 파이프라인 그대로). 과거 신탁 이력 표시

### 3. 러너 상시화 (WO-010 제안 ⑴ 채택)

- Windows 작업 스케줄러 "로그온 시 시작" 등록 스크립트 + 등록 실행. 중복 가드로 수동 기동과 공존. 뷰어 서버도 동일하게 등록 (선택 플래그)
