# WO-004: 검수 반영 + 사이클 2·3 실행

## 배경

사이클 1 검수 통과. 실행 중 드러난 규칙 공백 8건에 대한 판정을 반영하고 사이클 2·3을 연속 실행한다.
QA 관찰 항목: ① 과잉 정합성(모든 사건이 주역 5인에 봉사하는 경향) ② 텐션 단조 증가(이완 밸브 부재).
이번 사이클에서 규칙으로 강제하지 않는다 — 엔진의 자연스러운 경향을 관찰하기 위함이니
엔진에게 이 관찰 항목을 알리지 말 것.

## 작업 범위

### 1. engine/rules.md 개정 (문구 그대로)

- 규칙 1 보강 (날짜 체계): "날짜는 기록력을 쓴다 — 연대기가 시작된 주가 기록력 1년 1주이며, 사이클 N = 기록력 N번째 주(연도는 52주 단위). 기록 이전의 과거는 상대 표기와 cycle: null."
- 규칙 4 보강 (도시급 판정): "도시급 판정 기준: 사건에 직접 연루된 대상(인물·관계·구역·세력)의 텐션 상위 2개의 합이 70 이상."
- 규칙 7 보강 (append-only 범위): "append-only는 timeline의 truth·public_knowledge 서술 내용에 적용된다. 상태 파일의 현재 상태 필드(텐션·관계·wish 등)는 갱신 가능하다. 스키마 필드의 추가·폐기는 사이클 중 엔진이 임의로 하지 않는다 — 작업지시서로만 한다."
- 규칙 8 보강 (scar 범위): "scar는 affected 전원의 의무가 아니다. 사건이 인물의 내면·신체·평판·관계에 지속되는 흔적을 남겼다고 판단될 때 필수다. 역사가는 명백한 누락만 반려한다."
- 규칙 11-1 신설 (wish): "wish는 { text, status, note } 구조를 갖는다. status는 open | fulfilled | lost. 성취되거나 좌절된 wish는 삭제하지 않고 status로 남긴다 — 소원의 이력도 비가역이다. 인물은 새 wish를 품을 수 있으나 open 상태는 인물당 1개다."
- 규칙 15-1 신설 (역사가 재량): "연대기의 형식 — 제목 체계, 부기, 부재 진술('특기할 일이 없었다') — 은 역사가의 재량이다."

### 2. 스키마 정비

- timeline: 전 사건에서 `tension_delta` 제거 (폐기). 시드 3건에 `grade` 소급: 창시 신화 = 3, warehouse-fire = 2, fishermen-lost = 1
- characters: wish를 규칙 11-1 구조로 마이그레이션. 기존 텍스트 → `text`, 전원 `status: "open"`. 예론 wish의 `note`: "자릿세 대치를 명분으로 또 유예되었다" — 사이클 1 보고의 '좌절 방향 이동'을 상태에 반영
- 로한→미라 관계 note 수정: "화재 후 대장간 재건 자금을 빌렸다" (WO-002 오기 정정)
- CLAUDE.md 갱신: chronicle 파일명 `cycle-NNN.md` 통일(§3 트리), §5 wish 구조 반영

### 3. 사이클 2·3 연속 실행

- 절차는 WO-003 §4와 동일 (엔진 → 역사가 검수 → 집필 → 커밋). 사이클마다 커밋 분리.
- 신탁 없음 (자율 사이클 유지)
- 사이클 3까지 실행 후 정지. 사이클 4는 실행하지 않는다 (QA 검수 후 WO-005에서 첫 신탁 주입 예정)
