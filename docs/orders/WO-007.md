# WO-007: 사전 정비 + 미르한 사이트 v1

## 배경

WO-006 검수 판정 반영 후, Astro 정적 사이트 v1을 구축한다. 핵심 원칙: 사이트는 기록 계층만
렌더링한다. truth는 절대 읽지 않는다. 독자가 보는 미르한은 연대기·신문·공표된 신탁이 전부이며,
진실은 저장소 안에만 있다.

## 작업 범위

### 1. 사전 정비 — 규칙 개정 (문구 그대로)

- 규칙 4-1의 "임계 초과(합 70)" 표현을 "합 70 이상"으로 통일 (규칙 4와 문구 일치)
- 규칙 4-2 신설: "해소의 판정: 사건에 의해 축의 텐션(연루 대상 상위 2개 합)이 70 미만으로 내려가면 해소다. 사적 잔불이 남거나 다른 곳으로 옮겨 가는 것은 결말과 양립한다."
- 규칙 7-2 신설: "secret은 현재 상태 필드로서 재서술할 수 있으나 삭제할 수 없다. 있었던 일은 문구가 바뀌어도 사라지지 않는다."
- 규칙 15-3 신설: "역사가는 신문 발행 전 truth 누출 여부만 검수한다. 문체·해석·소문은 검수 대상이 아니다."
- 규칙 16-1 신설: "기록 계층의 채색(물가, 날씨의 감촉 등)은 세계의 사실이 아니다. 엔진은 다음 사이클에서 이를 채택해 truth로 승격할 수 있다."
- 규칙 19 신설: "신문 지면의 호수는 발행 순번(제N호)이며, 파일명은 사이클 번호를 따른다. 신문은 기존 연대기와 기존 신문을 읽을 수 있다."

### 2. 사전 정비 — 스키마

- `world/axes.json` 신설 (갈등 축 레지스트리, 엔진이 유지): `{ id, name, parties[], note, status(open|resolved), over_threshold_since(cycle|null) }` 초기값: 현재 열린 축들을 사이클 6 보고 기준으로 등재 (신전-갯벌, 이센-로한 포함, 해소된 축 2개는 status: resolved로 이력 보존)
- factions.json에 `kind` 필드 (`faction | institution`). 기존 2개는 faction, '부두 소식'을 institution으로 등재 (note: 발행인 미상 — world 진실로는 무명 필경사)
- characters.json에 `public` 서브객체 신설: `{ intro, known_details[] }`
  - 기입 원칙: chronicle/과 records/에 이미 적힌 사실만 담는다. 비밀·숨은 욕망·텐션·wish·scars에서 유래한 정보는 공개 기록에 이미 실렸더라도 그 실린 문구 수준까지만. 6명 전원 작성하고 완료 보고에 전문 첨부 (QA가 누출 검수)
  - 사이클 7부터는 엔진이 유지
- CLAUDE.md §3 트리에 axes.json 반영

### 3. 사이트 v1 구축 (site/, Astro)

데이터 접근 원칙 (완료 기준에서 기계 검증): 빌드가 읽을 수 있는 것 — `chronicle/`, `records/`,
`world/oracles.json`의 `proclaimed: true` 항목, `world/characters.json`의
`name·role·district·status·public`, `world/places.json`의 구역 이름·설명.
그 외 필드(tension, truth, secret, desires, wish, scars, relationships)는 import조차 금지.

페이지 5면:

1. 오늘의 소식 (메인 `/`) — 최신 신문 호를 지면처럼 렌더링. 지면 구획(1면/뒷면 단신/떠도는 말/부고/날씨와 바다)을 시각 구획으로. 신문이 없던 1~5주로 이동 시 "이 주에는 소식지가 없었다 — 연대기만 남아 있다" 안내와 연대기 링크
2. 연대기 (`/chronicle`) — 주차별 아카이브. 목록 + 개별 페이지. 기록실 명의 명시
3. 인물 도감 (`/people`) — 카드 그리드: 이름, 직업, 구역, 생존 여부, public.intro, public.known_details, 등장한 연대기·신문 링크(파일 내 이름 언급 기준 자동 수집). 사망자는 카드에 명확한 표식 (아직 없음)
4. 신탁의 기록 (`/oracles`) — proclaimed된 신탁만, 공표 사이클과 decree 원문
5. 지도 (`/map`) — v1은 자리만: 구역 3개의 이름·설명 텍스트 카드 + "지도는 측량 중" 문구. 인터랙티브 SVG는 후속 WO

디자인 브리프 (기획 방향 — 세부는 frontend-design 재량):

- 콘셉트: "발굴된 사료를 읽는다". 종이·잉크의 질감, 명조/세리프 계열, 절제된 장식
- 연대기와 신문은 문체만큼 지면도 달라야 한다 — 연대기는 여백 많고 단정하게, 신문은 조밀하고 활자가 성급하게
- 초상화·인물 일러스트 금지 (기존 결정). 색은 잉크 2~3색 이내
- 반응형 필수 (운영자는 폰으로 본다)

빌드: 저장소 push 시 GitHub Actions로 Astro 빌드 → GitHub Pages 배포.
(사이클 자동 실행은 WO-008 — 이 워크플로는 빌드·배포만 한다)
