# WO-008: 분리 배포 + CI 게이트 + 사이클 자동화

## 배경

사이트 v1 검수 통과. 배포는 저장소 분리 방식으로 확정: 세계 저장소(truth 포함)는 private 유지,
빌드 산출물(dist/)만 별도 public 저장소로 push하여 Pages 서빙. 이 WO가 끝나면 세계는 매일
스스로 한 사이클을 돌고, 사이트가 자동 갱신된다 — 1단계의 완성. QA 리듬은 사이클 단위에서
주간 단위로 전환된다.

## 운영자 사전 액션 (Claude Code 실행 전 필요)

1. 사이트용 public 저장소 생성 (이름 운영자 결정, 이하 `<SITE_REPO>`)
2. 세계 저장소 Actions secrets 등록:
   - `SITE_DEPLOY_TOKEN`: `<SITE_REPO>` push 권한의 fine-grained PAT
   - `ANTHROPIC_API_KEY`: 사이클 자동 실행용
3. WO-007 잔여: `gh auth refresh -s workflow` 후 세계 저장소 push

## 작업 범위

### 1. 도감 노출 정책 (WO-007 검수 판정)

- 카드 노출 조건: 인물의 이름이 chronicle/ 또는 records/ 텍스트에 1회 이상 등장. 빌드 시 자동 판정 (기존 등장 기록 링크 수집 로직 재사용)
- 현재 효과: 타루·이센 카드 비노출 (기록에 이름이 오르는 순간 자동 등장)
- axes.json에 예론×칼라 축 소급 등재 (status: resolved, 해소 사이클 5 명기)

### 2. 분리 배포

- 배포 워크플로 수정: Astro 빌드 → 검증 게이트(§3) 통과 시 → dist/를 `<SITE_REPO>`의 `gh-pages` 브랜치로 push (SITE_DEPLOY_TOKEN 사용)
- `<SITE_REPO>`에는 dist 산출물 외 어떤 파일도 push하지 않는다 (README 1장 예외 — "이 저장소는 미르한 사이트의 빌드 산출물입니다. 원본은 비공개" 수준)
- 세계 저장소는 private 유지. Pages는 `<SITE_REPO>`에서 활성화

### 3. CI 검증 게이트 (배포 전 필수 통과, 실패 시 배포 중단)

1. site/src 금지 필드 grep (WO-007 §3 원칙)
2. dist/ truth 문구 스캔 — WO-007의 7종을 시작으로, truth 전용 고유 문구 목록을 `engine/truth-canary.txt`로 분리 관리 (엔진이 매 사이클 새 비밀·truth 서술의 식별 문구를 1~3개씩 추가). 스캔 대상 문구가 사이클과 함께 자라는 구조
3. world/ JSON 파싱 + event_id 참조 무결성

### 4. 사이클 자동화 — `engine/run-cycle.mjs`

Anthropic API 호출 파이프라인 (모델 선택은 재량, 완료 보고에 명시):

1. 입력 준비: world/ 전체 + rules.md + (있으면) 미처리 신탁. 신문용 public_knowledge 추출은 별도 단계 (truth 격리 — WO-006 수동 절차의 스크립트화)
2. 엔진 실행 → world/ 갱신 + 사이클 보고를 `docs/reports/cycle-NNN.md`로 저장 (지금까지 완료 보고에 실리던 전문이 저장소에 남는 구조로 전환)
3. 역사가 실행: 검수 → 반려 시 엔진 재생성 1회 → 재반려 시 기록 소실 처리 (규칙 17: 해당 사이클은 "기록이 소실된 날" 커밋 + 보고에 사유) → 통과 시 집필
4. 신문 실행 (public 추출본만 입력) → 역사가 누출 검수 (규칙 15-3)
5. 기계 검증: JSON 파싱, 참조 무결성, truth-canary 스캔 (chronicle·records 대상)
6. 커밋·push → 배포 워크플로 트리거

- 크론: 매일 1회, KST 05:00 (운영자가 아침에 새 호를 본다)
- 실패(API 오류 포함) 시: 커밋 없이 종료 + Actions 실패 알림. 다음 날 크론이 이어서 실행 (사이클 번호는 실행 순번이므로 결번 없음 — 규칙 1과 정합)

### 5. 주간 QA 다이제스트

- 매주 1회 (일요일), `docs/qa/week-NN.md` 자동 생성: 그 주 사이클 보고 전문 연결 + 텐션 지형 변화 요약표 + 엔진·역사가·신문의 제안(규칙 18) 취합
- 운영자는 이 파일을 QA 세션(클로드)으로 가져온다 — 주간 검수 리듬의 반입물
