# WO-012: 그래픽 v2 — 사람이 사는 마을처럼

## 배경

뷰어 v1 가동 확인. 운영자 피드백: 탑뷰는 유지하되 디테일 상향 — 캐릭터는 동그라미가 아니라 사람 형태여야 하고, 마을도 더 마을다워야 한다. 이 WO는 그래픽 전면 개편 + WO-011 검수에서 채택된 사전 정비 3건을 포함한다.

WO-011 잔여 3건(routines.json 전문 제출, 자연 이벤트 자동 재생 확인, 스크린샷)은 이 WO 완료 보고에 함께 담는다.

## 작업 범위

### 1. 사전 정비 (WO-011 검수 채택분)

- staging 장소 어휘 규칙화: rules.md에 표준 장소 슬러그 11곳(신전·구시가·대장간·사과나무 마당·빵집·시장·항구 골목·예론의 오두막·부두·소금밭·갯벌 움막)을 명시. 엔진은 이 어휘만 사용, 새 장소가 필요하면 제안으로
- weather 진실화: 러너 출력에 `weather` 필드 추가 (이벤트 시점의 날씨 — 맑음/흐림/비/바람/안개/눈, 계절과 정합). 상단 바·지도 톤이 이를 읽고, 로한의 점심 루틴을 원래 조건("비 오는 날마다")으로 복원
- home 필드 정식화 (characters.json — 기획 확정, 아래 그대로): 로한 = 대장간 딸린 살림집 / 미라 = 빵집 이층 살림방 / 예론 = 부두 곁 오두막 / 이센 = 신전 기록실 곁방 / 칼라 = 항구 골목 셋집 / 타루 = 빵집 화덕 뒤 짚자리. 뷰어 임의 배치를 이 정본으로 대체

### 2. 캐릭터 스프라이트 — 사람 형태

스타일 기준 (택1, A 우선):

- A트랙: 운영자의 탑다운슈터 캐릭터 에셋 이식 — 운영자가 경로를 제공하면 그 스타일 (비율·해상도·팔레트 감각)을 기준으로 마을 복장 변형 제작
- B트랙 (경로 미제공 시): 자체 제작 — 탑뷰 픽셀 휴머노이드, 머리·몸통·팔다리가 구분되는 실루엣. 스프라이트는 코드 생성이든 수제든 무방하나 6명 + 무명 배우가 한 스타일이어야 함

요구 사항:

- 4방향(상하좌우) + 걷기 애니메이션(2~4프레임) + 정지 자세. 취침은 눕기 또는 지붕 아래 💤 유지
- 인물 식별이 실루엣·색·소품으로 가능할 것 (기획 지정): 로한 = 가죽 앞치마·넓은 어깨 / 미라 = 앞치마·목에 건 조개껍데기(흰 점 하나면 충분) / 예론 = 챙 모자·굽은 등·지팡이 노 / 이센 = 어두운 두루마기·손에 두루마리 / 칼라 = 단정한 옷·허리의 장부 / 타루 = 작은 키·맨발. 무명 배우 = 회색 제네릭 휴머노이드 + 역할 라벨
- 이름표는 유지 (스프라이트 아래, 기존보다 작게)

### 3. 마을 타일 업그레이드

- 건물: 단색 rect → 지붕·벽·문이 구분되는 탑뷰 건물 (11곳 각자 개성 — 신전은 돌, 대장간은 굴뚝 연기, 빵집도 굴뚝, 오두막은 판자)
- 지형: 길(건물 사이 자연 연결), 사과나무 마당의 나무 1그루(로한 서사의 성지 — 공들일 것), 부두 판자, 소금밭 격자, 바다 물결(은은한 애니메이션)
- 밤: 야간 오버레이 + 창불 유지, 등불 몇 개 추가. 날씨 연출: 비(빗줄기), 눈, 안개(채도)
- 성능: 상시 켜두는 화면이므로 CPU 점유 낮게 (requestAnimationFrame 절제, 탭 비활성 시 애니메이션 정지)
