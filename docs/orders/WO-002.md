# WO-002: 스키마 보강 + 시드 보정 + 규칙서 v0

## 배경

WO-001 검수 결과 반영. Claude Code의 제안 3건(district, relationships 분리, scars 소급)이
채택되었고, 기획 측 누락 1건(어부 실종 사건)과 시간 규정 미비 1건이 확인되었다.
이 WO는 그 보정과 함께 규칙서 v0를 배치한다. **규칙서 전문은 §4에 제공됨 — 내용 수정 금지.**
프롬프트(world-engine.md, historian.md)는 WO-003.

## 작업 범위

### 1. 시간 규정 반영 (기획 확정)

- **1사이클 = 세계 시간 7일.** 사이클 번호는 엔진 실행 순번(1부터 시작).
- 사이클 1 이전의 모든 과거 사건: `cycle: null`, `date`가 정본. 음수 사이클 폐지.
- timeline.json의 창고 대화재를 `cycle: null, date: "10년 전"`으로 수정.

### 2. characters.json 스키마 보강 및 시드 보정

- 전원에 `district` 추가: 로한·미라·예론·칼라 = 항구 구역, 이센 = 언덕 구역
- 칼라에 `origin_district: 갯벌 구역` 추가 (선택 필드)
- relationships를 `type`(표준 어휘) + `note`(자유 서술) + `tension`으로 재구성.
  표준 어휘: `가족 | 채무 | 거래 | 우정 | 연정 | 원한 | 갈등 | 은혜 | 비밀공유`
  - 로한→미라: type 채무, note "빵집 재건 자금을 빌렸다"
  - 미라→로한: type 채무, note "화재 후 대장간 일감을 몰아준 채권자"
  - 예론↔칼라: type 갈등, note "부두 자릿세 인상을 둘러싼 다툼"
  - 이센→로한: type 갈등, note "화재 증언 기록을 요청했다 거절당했다"
- scars 소급 기입 (기획 확정 — 문구 그대로):
  - 로한: `{ "cycle": null, "event": "창고 대화재", "mark": "그날 본 것을 누구에게도 말하지 못했다. 화재 종소리를 들으면 손이 떨린다" }`
  - 미라: `{ "cycle": null, "event": "창고 대화재", "mark": "동생이 실종되었다. 연기 냄새를 맡으면 하던 일을 멈춘다" }`
  - 예론: `{ "cycle": null, "event": "어부 실종", "mark": "폭풍 전조를 보고도 침묵했다. 그날 이후 밤바다를 보지 않는다" }`

### 3. timeline.json 시드 사건 추가 (기획 확정)

어부 실종 사건:

```json
{
  "id": "fishermen-lost",
  "cycle": null,
  "date": "1개월 전",
  "public_knowledge": "어부 3명이 밤 조업 중 실종되었다. 부서진 뱃조각만 떠올랐다. 부두에는 바다 괴물 소문이 돌기 시작했다.",
  "truth": "돌풍에 의한 조난. 예론은 그날 저녁 폭풍의 전조를 읽었으나 아무에게도 알리지 않았다.",
  "affected": ["yeron-the-fisher"],
  "tension_delta": 0
}
```

(tension_delta 0 — 현재 텐션 값에 이미 반영된 것으로 간주, WO-001 처리 방식과 동일)

### 4. engine/rules.md v0 생성 (전문 — 그대로 배치)

```markdown
# 규칙서 v0 — 세계 운영 규칙 (모든 에이전트 공통)

## 시간
1. 1사이클 = 세계 시간 7일. 사이클 번호는 엔진 실행 순번, 세계 날짜는 date가 정본.
2. 사이클 1 이전의 과거는 cycle: null. 음수 사이클은 존재하지 않는다.

## 사건 생성 (세계 엔진)
3. 목표는 재미가 아니라 지속이다: 오늘 하루도 세계가 살아가게 하라. 드라마를 만들지 마라.
4. 사건 규모 등급: 일상(0) / 지역(1) / 도시(2) / 시대(3). 기본값은 일상~지역.
   도시급은 관련 텐션 합이 70 이상일 때만, 시대급은 신의 개입 또는 텐션 90 이상에서만.
5. 사이클당 사건 1~3개. 도시급 이상은 사이클당 최대 1개.
6. 사건은 가장 높은 텐션 지점에서 분기한다. 매 사이클 최소 1개의 갈등은 미해결로 남긴다.
7. 사망·파괴는 비가역. truth는 append-only — 과거의 진실은 수정하지 않는다.
8. 사건이 인물을 통과하면 반드시 scar를 남긴다.
9. 인물의 비밀·욕망이 참조하는 사건은 timeline에 존재해야 한다.

## 인물
10. 주인공은 없다. 서사적 비중은 사건이 결정한다.
11. 새 인물은 quirks 2개 이상, wish 1개 필수. 실존 인물 상한 30명, 초과 시 생성 대신 배경 인구 활용.
12. 배경 인구는 수치로만 움직인다. 이름이 생기는 순간 실존 인물이다.
13. 관계 type은 표준 어휘(가족|채무|거래|우정|연정|원한|갈등|은혜|비밀공유)만 사용.
    새 type이 필요하면 사용하지 말고 완료 보고에 제안으로 남긴다.

## 기록 (역사가 이하 모든 기록 계층)
14. 세계 엔진만 world/를 쓴다. 역사가는 world/를 읽고 chronicle/에만 쓴다.
15. 역사가는 truth를 그대로 옮기지 않는다. 공식 기록은 "공식적으로 알려진 만큼"만 담으며
    편향과 오류가 허용된다. 단, 기록이 truth와 다를 경우 그 차이는 의도된 것이어야 한다.
16. 모르는 것은 모른다고 기록한다. "원인은 기록되지 않았다"는 완성된 문장이다.
17. 사이클 실패 시 그날은 "기록이 소실된 날"이다. 실패도 세계관으로 흡수한다.
```
