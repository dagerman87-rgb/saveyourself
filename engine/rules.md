# 규칙서 v0 — 세계 운영 규칙 (모든 에이전트 공통)

## 시간
1. 1사이클 = 세계 시간 7일. 사이클 번호는 엔진 실행 순번, 세계 날짜는 date가 정본.
   날짜는 기록력을 쓴다 — 연대기가 시작된 주가 기록력 1년 1주이며, 사이클 N = 기록력 N번째 주
   (연도는 52주 단위). 기록 이전의 과거는 상대 표기와 cycle: null.
2. 사이클 1 이전의 과거는 cycle: null. 음수 사이클은 존재하지 않는다.

## 사건 생성 (세계 엔진)
3. 목표는 재미가 아니라 지속이다: 오늘 하루도 세계가 살아가게 하라. 드라마를 만들지 마라.
4. 사건 규모 등급: 일상(0) / 지역(1) / 도시(2) / 시대(3). 기본값은 일상~지역.
   도시급은 관련 텐션 합이 70 이상일 때만, 시대급은 신의 개입 또는 텐션 90 이상에서만.
   도시급 판정 기준: 사건에 직접 연루된 대상(인물·관계·구역·세력)의 텐션 상위 2개의 합이 70 이상.
5. 사이클당 사건 1~3개. 도시급 이상은 사이클당 최대 1개.
6. 사건은 가장 높은 텐션 지점에서 분기한다. 매 사이클 최소 1개의 갈등은 미해결로 남긴다.
7. 사망·파괴는 비가역. truth는 append-only — 과거의 진실은 수정하지 않는다.
   append-only는 timeline의 truth·public_knowledge 서술 내용에 적용된다. 상태 파일의 현재 상태
   필드(텐션·관계·wish 등)는 갱신 가능하다. 스키마 필드의 추가·폐기는 사이클 중 엔진이 임의로
   하지 않는다 — 작업지시서로만 한다.
8. 사건이 인물을 통과하면 반드시 scar를 남긴다. scar는 timeline 사건의 event_id를 참조한다.
   scar는 affected 전원의 의무가 아니다. 사건이 인물의 내면·신체·평판·관계에 지속되는 흔적을
   남겼다고 판단될 때 필수다. 역사가는 명백한 누락만 반려한다.
9. 인물의 비밀·욕망이 참조하는 사건은 timeline에 존재해야 한다.

## 인물
10. 주인공은 없다. 서사적 비중은 사건이 결정한다.
11. 새 인물은 quirks 2개 이상, wish 1개 필수. 실존 인물 상한 30명, 초과 시 생성 대신 배경 인구 활용.
11-1. wish는 { text, status, note } 구조를 갖는다. status는 open | fulfilled | lost.
    성취되거나 좌절된 wish는 삭제하지 않고 status로 남긴다 — 소원의 이력도 비가역이다.
    인물은 새 wish를 품을 수 있으나 open 상태는 인물당 1개다.
12. 배경 인구는 수치로만 움직인다. 이름이 생기는 순간 실존 인물이다.
13. 관계 type은 표준 어휘(가족|채무|거래|우정|연정|원한|갈등|은혜|비밀공유)만 사용.
    새 type이 필요하면 사용하지 말고 완료 보고에 제안으로 남긴다.
    관계 note는 소유자 시점에서 상대와의 관계를 서술한다.
13-1. 관계는 소유자의 인식이다. 단방향 관계를 허용하며, 양방향 관계의 tension이 서로 다른 것도
    허용한다 — 인식의 비대칭은 세계의 질감이다.

## 기록 (역사가 이하 모든 기록 계층)
14. 세계 엔진만 world/를 쓴다. 역사가는 world/를 읽고 chronicle/에만 쓴다.
15. 역사가는 truth를 그대로 옮기지 않는다. 공식 기록은 "공식적으로 알려진 만큼"만 담으며
    편향과 오류가 허용된다. 단, 기록이 truth와 다를 경우 그 차이는 의도된 것이어야 한다.
15-1. 연대기의 형식 — 제목 체계, 부기, 부재 진술("특기할 일이 없었다") — 은 역사가의 재량이다.
16. 모르는 것은 모른다고 기록한다. "원인은 기록되지 않았다"는 완성된 문장이다.
17. 사이클 실패 시 그날은 "기록이 소실된 날"이다. 실패도 세계관으로 흡수한다.
